Про мене

Junior Business Analyst із фінансово-економічною освітою та завершеним навчанням з Business Analysis. Розвиваю практичні навички бізнес-аналізу через роботу над портфоліо, яке охоплює аналіз вимог, управління стейкхолдерами, моделювання бізнес-процесів і підготовку документації.

Відзначаюся аналітичним мисленням та уважністю до деталей. Люблю глибоко розбиратися в процесах, причинах виникнення проблем і взаємозв'язках між бізнес-потребами та технічними рішеннями. Швидко навчаюся, відкритий до зворотного зв'язку та прагну переймати досвід більш досвідчених колег.

Технічні навички

BPMN / UML

Розумію основи та застосовую для моделювання бізнес-процесів та сценаріїв взаємодії. Створюю Use Case та Activity Diagram.

User Stories / AC

Практикував створення та документування вимог. Розробляю User Stories, Acceptance Criteria з використанням BDD-підходу.

SQL

Базовий рівень. Працюю з SELECT, JOIN, WHERE, GROUP BY. Розумію застосування SQL у бізнес-аналітиці.

API

Розумію принципи роботи API. Мав досвід створення API-документації для портфоліо.

Jira / Confluence

Володію базовим розумінням для управління вимогами, задачами та документацією.

Agile (Scrum / Kanban)

Розумію основні етапи SDLC та принципи роботи Agile-команд. Знайомий з RACI та Communication Planning.

Практичний аналіз процесу оформлення замовлення та оплати електронної книги. Включає Use Case Specification з основним і альтернативними потоками, Use Case Diagram, Activity Diagram.

Документи: User Stories, DoR, DoD | Формат AC: Gherkin

Breadcrumb Navigation, Delete Account Confirmation Modal, Promo Code. Кожна User Story містить Acceptance Criteria, написані з використанням BDD-підходу (Given / When / Then).

Документи: Stakeholder List, Matrix, RACI + SDLC, Communication Plan

Покривають повний цикл аналізу стейкхолдерів: ідентифікація → вплив / зацікавленість → ролі й відповідальність → план комунікації. Трассування ролей через призму SDLC і Scrum-процесу.
